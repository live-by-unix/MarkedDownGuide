## History
Markdown was created to make writing for the web easier.           
Before, you had to choose between writing:      
* Raw [HTML](https://en.wikipedia.org/wiki/HTML), which took a long time.
* Or [WYSIWYG](https://en.wikipedia.org/wiki/WYSIWYG) ("What you see is what you get"), which made messy HTML.

Software dev [John Gruber](https://en.wikipedia.org/wiki/John_Gruber) and [Aaron Swartz](https://en.wikipedia.org/wiki/Aaron_Swartz) collabed to create the first [Markdown](https://en.wikipedia.org/wiki/Markdown) language & interpreter, which was originally written in [Perl](https://en.wikipedia.org/wiki/Perl).           

Markdown was made in 2004.     
It included basic things, like:
* Headings
* Lists
* Links
* Code blocks
* Block Quotes
* Images
* **And more!**

Then, a few years after release, [Markdown](https://en.wikipedia.org/wiki/Markdown), becomes very popular.        

In 2010, extensions of markdown appear, adding things like:   
* Tables
* Fenced code blocks
* Task lists
* Strikethrough
* Emoji support :smiley:
* **And more!**

Then, in 2014, the [CommonMark Project](https://en.wikipedia.org/wiki/Markdown#CommonMark) project is started to create a uniform [Markdown](https://en.wikipedia.org/wiki/Markdown).     

Today:
Now, Markdown is used **Everywhere!**

### Headings
The six heading levels are:
```markdown
# Heading Level 1 (Main Title / <h1>)
## Heading Level 2 (Major Section / <h2>)
### Heading Level 3 (Subsection / <h3>)
#### Heading Level 4 (Sub-subsection / <h4>)
##### Heading Level 5 (Rarely Used / <h5>)
###### Heading Level 6 (Smallest / <h6>)
``` 
These are the common ways to use the different headings. But of course, you can use them in different ways.   

### Lists

#### Unordered lists
These are lists which are like unordered bullet points in GDocs.          
You can use `-` (minus) , `+` (plus) , or `*`(asterisk)
For example:    
* This
* is
* a
* list

- This
- is
- a
- list

and     
+ This
+ is
+ a
+ list

The one I like the best are asterisks.
